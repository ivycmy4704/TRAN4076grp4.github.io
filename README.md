# ProGen AI - Professional Product Image Generator  
**User & Administrator Manual**  
**Version 1.0** – November 28, 2025

### Part 1: Client Manual

**Welcome to ProGen AI**  
Your professional tool for instantly generating high-quality product images using advanced AI.

#### 1.1 Getting Started
- **Website**: https://ivycmy4704.github.io/langtech_testing/
- **Supported browsers**: Chrome, Firefox, Safari (version 100 or newer)
- JavaScript must be enabled
- A registered account is required to generate images

#### 1.2 Registration
1. On the login page, click **“Don’t have an account? Create Account”**
2. Enter your email address
3. Create a strong password (8+ characters, letters + numbers + symbols)
4. Click **“Create Account & Send Verification Code”**
5. Check your inbox (including spam) for the 6-digit code
6. Enter the code to complete registration
7. You may now log in

**Tip**: Use a password manager and check spam if the email doesn’t arrive.

#### 1.3 Logging In
1. Enter your email and password
2. Click **“Log In”**
3. You will be taken to your dashboard

**Troubleshooting**  
• Wrong credentials → double-check spelling and caps lock  
• Forgot password → click **“Forgot Password?”** (see 1.4)

#### 1.4 Password Recovery
1. Click **“Forgot Password?”**
2. Enter your registered email
3. Click **“Send Verification Code”**
4. Enter the code received via email
5. Set and confirm a new password

#### 1.5 Using the Image Generation Tool
1. From the dashboard, go to **Generate Image**
2. Type a detailed product description  
   Example: “A sleek black wireless earbuds on a pure white background, studio lighting, 45-degree angle”
3. (Optional) Upload a reference image or choose style settings
4. Select resolution (e.g., 1024×1024) and aspect ratio
5. Click **Generate** (takes 10–60 seconds)
6. Preview → edit prompt and regenerate if needed
7. Download the final image (PNG or JPG)

**History**  
All past generations are saved in **“View Past History”** on the Home page.

**Free-tier limit**  
20 generations per account. Purchase more credits via the **Credits System** on the Home page.

**Best Practices**  
- Specify lighting, angle, background, materials, and colors  
- Start simple, then refine the prompt

#### 1.6 Account Management
- Update email/password → Settings menu
- Log out → Profile icon → **Log Out**
- Need help → Use the **Support dropbox** on the Home page

#### 1.7 Client Troubleshooting
| Issue                  | Solution                                      |
|------------------------|-----------------------------------------------|
| Login fails            | Check credentials; clear browser cache        |
| No verification email  | Check spam; click “Resend code”               |
| Generation errors      | Stable internet; simplify prompt              |
| Slow loading           | Try incognito mode or another browser         |

Additional help is available on the **Support** page.

### Part 2: Admin Manual  
(For platform administrators only – special credentials required)

#### 2.1 Admin Access
Log in using the standard login page with your admin email/username and password (e.g., admin / admin001).

#### 2.2 Admin Dashboard
- Create premium templates (name, industry, aesthetic/art style, colors, etc.)
- Assign templates to clients by entering their emails (one per line)
- Save & generate code
- View all templates and assignments

#### 2.3 Managing Users (Client Management tab)
- Search/filter users by email or company
- View profile, generation count, history, last login
- Edit company name, reset credits to 20, suspend, or delete accounts
- Bulk actions available

#### 2.4 Client Support tab
- View support tickets and recent login history

#### 2.5 Monitoring & Reports
- Activity logs (filterable, exportable to CSV)
- Usage reports & error logs

#### 2.6 System Configuration
- Adjust rate limits and quotas
- Configure SMTP for emails
- Schedule backups (Maintenance section)

#### 2.7 Security & Maintenance
- All admin actions are logged
- Check for updates on the GitHub repository
- Always log out from shared devices

#### 2.8 Admin Troubleshooting
| Issue                | Solution                              |
|----------------------|---------------------------------------|
| Access denied        | Verify admin role; contact developer  |
| Logs not loading     | Clear cache; check server status     |
| User edit fails      | Ensure no active sessions; retry      |
| Export error         | Use smaller date range                |

For advanced issues, review the source code on GitHub.
